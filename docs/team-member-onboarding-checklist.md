# Onboarding Checklist for New Team Members

This checklist helps new team members understand their role, learn about key processes, and integrate quickly into the OctoAcme organization.

## Week 1: Orientation

### Day 1: Welcome & Setup
- [ ] Complete company/org onboarding
- [ ] Receive welcome packet with key information
- [ ] Set up development environment and tools
  - [ ] VPN and network access
  - [ ] GitHub/version control access
  - [ ] Communication tools (Slack, Teams, etc.)
  - [ ] Collaboration tools (Jira, Confluence, etc.)
  - [ ] Code editors and development tools
- [ ] Meet direct manager/mentor
- [ ] Schedule 1:1 meetings with key cross-functional partners
- [ ] Create personal development plan

### Day 2-3: Team Onboarding
- [ ] Attend team welcome meeting
- [ ] Meet all team members in your department
- [ ] Understand current projects and priorities
- [ ] Review recent project documentation
- [ ] Get tour of relevant systems and dashboards

### Day 4-5: Role-Specific Onboarding
- [ ] **For Developers**:
  - [ ] Clone and build main repository
  - [ ] Run test suite and verify setup
  - [ ] Review code organization and naming conventions
  - [ ] Complete code review training
  - [ ] Make first pull request (documentation or small fix)

- [ ] **For Product Managers**:
  - [ ] Review product roadmap and vision
  - [ ] Understand customer segments and use cases
  - [ ] Review recent product decisions and rationale
  - [ ] Meet key stakeholders (executives, customers, support)
  - [ ] Understand metrics and KPIs tracked

- [ ] **For Project Managers**:
  - [ ] Review current project plans and timelines
  - [ ] Understand dependency map and critical path
  - [ ] Meet all cross-functional project leads
  - [ ] Review risk register and mitigation plans
  - [ ] Understand communication cadence and status reporting

- [ ] **For UX/UI Designers**:
  - [ ] Review design system and component library
  - [ ] Understand product vision and user personas
  - [ ] Review recent design decisions
  - [ ] Access design tools and collaborate on sample design
  - [ ] Meet Product and Development teams

- [ ] **For DevOps Engineers**:
  - [ ] Review infrastructure architecture and diagrams
  - [ ] Understand deployment procedures and runbooks
  - [ ] Access production systems (read-only initially)
  - [ ] Review monitoring and alerting setup
  - [ ] Understand incident response procedures

- [ ] **For Business Analysts**:
  - [ ] Review recent requirements and specifications
  - [ ] Understand business processes and workflows
  - [ ] Meet stakeholders and business teams
  - [ ] Understand traceability and documentation standards
  - [ ] Review past projects and lessons learned

- [ ] **For Scrum Masters**:
  - [ ] Attend current sprint ceremonies
  - [ ] Understand team velocity and sprint metrics
  - [ ] Review team retrospectives and action items
  - [ ] Understand team dynamics and known issues
  - [ ] Review retrospective results and improvements made

- [ ] **For Customer Support Leads**:
  - [ ] Review common support issues and FAQ
  - [ ] Understand support tools and workflows
  - [ ] Meet support team members
  - [ ] Review recent customer feedback and trends
  - [ ] Understand escalation procedures

## Week 2: Deep Dive

### Cross-Functional Understanding
- [ ] Read [OctoAcme Personas](./octoacme-roles-and-personas.md) document
- [ ] Understand your role and how it interacts with other personas
- [ ] Read [Cross-Functional Collaboration Guide](./cross-functional-collaboration-guide.md)
- [ ] Schedule 1:1 meetings with key collaborators in other functions
- [ ] Attend cross-functional meeting with your department

### Process & Documentation
- [ ] Review project management processes
- [ ] Understand current tools and systems
- [ ] Access and understand documentation structure
- [ ] Review [Process Improvement Checklist](./process-improvement-checklist.md) to understand how processes evolve
- [ ] Understand decision-making and escalation procedures

### Project Deep Dive
- [ ] Understand current project status and timeline
- [ ] Attend project status meeting
- [ ] Review requirements and scope documentation
- [ ] Understand success metrics and acceptance criteria
- [ ] Schedule technical/product deep dive if applicable

### System & Tool Training
- [ ] Complete training on relevant systems:
  - [ ] Version control system (Git/GitHub)
  - [ ] Issue tracking (Jira, GitHub Issues)
  - [ ] Documentation platform (Confluence, Wiki)
  - [ ] Communication tools (Slack, Teams)
  - [ ] CI/CD system (relevant for dev/DevOps)
  - [ ] Monitoring/observability (relevant for DevOps/QA)
- [ ] Ask questions and get hands-on practice

## Week 3: Integration & First Contributions

### First Tasks/Projects
- [ ] Complete onboarding tasks assigned by manager
- [ ] Make first meaningful contribution to project
- [ ] Participate actively in team meetings
- [ ] Conduct code review or technical review (for relevant roles)
- [ ] Provide feedback on onboarding experience

### Relationship Building
- [ ] Schedule coffee chats with team members
- [ ] Participate in team social activities
- [ ] Join relevant Slack/Teams channels
- [ ] Understand team culture and values
- [ ] Build relationships across departments

### Check-in & Feedback
- [ ] Meet with manager for first month check-in
- [ ] Share feedback on onboarding process
- [ ] Discuss progress on personal development plan
- [ ] Identify any gaps or additional training needed
- [ ] Establish ongoing 1:1 meeting cadence

## Month 2-3: Ramping Up

### Independent Work
- [ ] Increase responsibility and project ownership
- [ ] Work independently on assigned tasks
- [ ] Conduct code/design/requirement reviews
- [ ] Participate in planning and estimation
- [ ] Mentor other new team members (if applicable)

### Continuous Learning
- [ ] Take relevant training courses
- [ ] Read recommended books/articles
- [ ] Attend conferences or webinars (if available)
- [ ] Learn from experienced team members
- [ ] Review lessons from past projects

### 30/60/90 Day Checkpoints
- [ ] **30 Days**: Completed onboarding, productive in assigned role
- [ ] **60 Days**: Successfully contributing to projects, integrated with team
- [ ] **90 Days**: Independent in role, established relationships, ready for more responsibility

### Feedback & Adjustments
- [ ] Provide feedback on onboarding effectiveness
- [ ] Discuss career goals and development areas
- [ ] Identify strengths and areas for growth
- [ ] Adjust role/responsibilities if needed
- [ ] Plan next learning objectives

## Ongoing: Staying Connected

### Regular Check-ins
- [ ] Weekly 1:1 with manager
- [ ] Monthly team meetings and planning
- [ ] Quarterly reviews and retrospectives
- [ ] Annual development plan review

### Continuous Improvement
- [ ] Suggest improvements to processes
- [ ] Share knowledge with team
- [ ] Mentor newer team members
- [ ] Participate in retrospectives and feedback
- [ ] Stay updated on organizational changes

## Onboarding Manager Checklist

- [ ] Prepare workspace and tools before arrival
- [ ] Create welcome packet with key information
- [ ] Schedule onboarding meetings and sessions
- [ ] Assign mentor/buddy for first few weeks
- [ ] Check in daily during first week
- [ ] Solicit feedback at 1-week, 2-week, and 1-month marks
- [ ] Ensure all access and permissions are granted
- [ ] Make introductions to cross-functional partners
- [ ] Schedule role-specific training
- [ ] Discuss goals and expectations
- [ ] Plan first meaningful project/task
- [ ] Conduct regular 1:1s
- [ ] Provide constructive feedback
- [ ] Celebrate first contributions and wins
- [ ] Support professional development
- [ ] Conduct 30/60/90 day reviews
